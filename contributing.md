# Contributing to ScaleDown

There are 4 different ways you can help us build and grow this package. All contributions are welcome and they are equally important in helping democratize education and access to TinyML!

## Request for Hardware

If you are new to the framework or to TinyML and want to learn about the package as well as help us improve it, then this is the best place to get started.

You can use our framework to build projects and then deploy models you build to a TinyML hardware. Request us for a hardware and we will send it to you!

We would love to see the stuff you are working and share it with the rest of the community! Doing so can also help the community by creating more resources to help beginners learn from. Below is how you can contribute projects.

## Write Content and Create Learning Resources

Once you have become comfortable with the package and have built a project, we would love to showcase it to our community!

You can also submit blogs and learning resources to help the community. These will be posted on our blog.

To contribute, please create a PR and we will review and approve it.

## Build the Package

After you have become very comfortable with the package, you can start to contribute to the development of the package.

The best way to do this is to first contribute to the documentation. This will help you understand the internals of the package.

Next, you can contribute small bug fixes and finally you can start to build features and enhancements. To do this, you can pick up an issue and start to work on it! Let us know that you are working on it by commenting on the issue and also joining our weekly standup! After you are ready to add your changes to the package, create a PR and we will approve it ASAP!
